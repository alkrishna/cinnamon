ExpectedConditions

global with sharing class ExpectedConditions
Description ExpectedConditions is a utility class that provides a list of canned ExpectedConditions which can be used when a test needs to wait for a certain condition to become true or false. ExpectedConditions are modeled after those in Selenium Webdriver ExpectedConditions class.

For example, if your test is looking for a certain element or text in a given page after making some action, you can use one of the ExpectedConditions method to make your test wait until the condition you specified turns true. See a sample code below. 


public class TestWithExpectedCondition extends cinnamon.BaseTest {
    private CustomPageObject customPage;

    public override void setUp(cinnamon.Context context) {
        customPage = (CustomPageObject) context.getPageObject(CustomPageObject.class);
        customPage.initializePageObject();
    }

    public override void test(cinnamon.Context context) {
        cinnamon.ElementLocator locator = new cinnamon.ElementLocator('div.title');
        customPage.fillForm();
        customPage.clickSave();

        // this condition waits until div.title element that contains the text 'Hello World' appears
        cinnamon.ExpectedCondition condition =
            cinnamon.ExpectedConditions.textToBePresentInElement(locator, 'Hello World!');
        customPage.getWait().until(condition);
    }
}
