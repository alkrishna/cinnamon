BaseTest

global abstract class BaseTest
Description BaseTest is an abstract base class that defines a single test in Cinnamon.

When a Cinnamon test runs, Cinnamon framework manages the entire lifecycle of a test execution and executes the following methods (setup, test and teardown) during the test execution. Context context must be used to store and retrieve test fixture used during test execution.


public class TestAccountPage extends BaseTest {

    // Create an account record as a test fixture
    public override void setUp(Context context) {
        Account acc = new Account();
        acc.name = 'account_' + System.currentTimeMillis();
        acc.site = 'San Francisco';
        insert acc;
        context.put('accId', acc.Id);
    }

    // Setting the URL to start for this test
    public override String getStartingPath(Context context) {
        String accId = (String) context.get('accId');
        return '/apex/EditAccount?id=' + accId;
    }

    // All test logic should be in this method
    public override void test(Context context) {
        String accId = (String) context.get('accId');

        AccountDetailPageObject accountPage = (AccountDetailPageObject)
                        context.getPageObject(AccountDetailPageObject.class);
        accountPage.typeAccountSiteField('New York');
        accountPage.clickSave();

        Account acc = [select Id, name, site from Account where Id = :accId];
        System.assert(acc.site.equalsIgnoreCase('New York'), 'site does not match');
    }

    // Clean up the account record at the end of test execution
    public override void tearDown(Context context) {
        List<ID> Ids = new List<ID>();
        Ids.add((String) context.get('accId'));
        Database.delete(Ids);
    }
}

global virtual ExpectedCondition getStartingPathExpectedCondition(Context context)
Description Returns an expected condition that will be evaluated during the test start up after we request the url determined by BaseTest.getStartingPath(). The base implementation is: 

return ExpectedConditions.visibilityOf(new ElementLocator('body.sfdcBody'));

This implementation works well for Visualforce pages that have simple page load events Test authors with more complex pages will benefit from overriding this method and providing a more specific condition that should be met before the test starts
